Comparing hashtable implementations performance by setting and accessing english words one by one

  • Array based hashtable with open addressing with precalculated array size
  • Linked List based hashtable with separate chaining
  • JS native Object
  • JS native Map
  • Rust WebAssembly Native HashMap (without any optimizations with direct string passing from JS to wasm)

JS object is basically a hashtable, but Map has some additional opimizations and should be faster.

In Rust WebAssembly HashMap, the biggest overhead probably comes from communitation - JS cannot direct pass a string, must encode it first.

Clearly Map is faster than any of other implementations
